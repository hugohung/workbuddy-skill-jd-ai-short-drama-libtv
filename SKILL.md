# libtv脚本及分镜生成助手

将「京东AI短剧剧本助手」产出的剧本文档上传至 LibTV 画布，自动完成项目创建、节点布局、脚本生成和分镜图准备。

## 触发条件

当用户提出以下类型请求时，使用此skill：
- 从剧本助手完成剧本后选择"进入libtv出片助手"
- 提及"libtv出片"、"生成分镜"、"上传剧本到libtv"等关键词
- 提供了京东AI短剧剧本助手格式的剧本文档（Markdown），要求导入 libtv

## 前置依赖

1. **剧本助手产出**：用户必须已通过「京东AI短剧剧本助手」生成完整剧本文档（Markdown 格式）
2. **LibTV CLI**：已安装 `libtv` 命令行工具（版本 ≥ 1.0.2），安装路径 `C:\Users\honghaoxiang\.libtv\libtv.exe`
3. **LibTV 登录**：用户必须已完成 `libtv login web` 或 `libtv login phone` 登录
4. **libtv-cli skill**：已安装 libtv-cli skill（`~/.workbuddy/skills/libtv-cli/`），所有 libtv 操作通过此 skill 的命令完成

## 重要提示

- **分镜图自动生成**：本 skill 使用 `libtv script storyboard` 命令自动生成分镜图（16:9 1K），分镜图费用较低，自动生成可节省用户操作
- **⛔ 严禁触发视频生成（红线）**：Agent 绝对禁止执行任何视频生成操作（包括 `libtv script video`、触发 video 节点、渲染导出等）。视频生成成本极高，必须由用户在 LibTV 画布中手动操作。违反此规则将导致严重损失
- **所有 libtv 操作通过 CLI 完成**：不要自己构造 HTTP 请求或绕到网页端操作，一律使用 `libtv` CLI 命令
- **每步告知用户当前进度**：与剧本助手一致，每个阶段开始前先告知用户

### 阶段1：解析剧本文档

> 📋 **第1步：解析剧本文档** — 读取 Skill 1 产出的剧本文档，提取各部分内容

**操作步骤**：
1. 读取用户提供的剧本文档（Markdown 文件路径，或从对话上下文中获取）
2. 按照文档结构提取以下部分的内容（纯文本，去除 Markdown 标记符号）：
   - **剧本正文**：第五部分「完整剧本正文」的全部内容
   - **人物设定**：第六部分「人物角色设定」的全部内容（含 AI 生成提示词）
   - **场景设定**：第七部分「场景设定」的全部内容（含 AI 生成提示词）
   - **静态封面提示词**：第八部分「静态封面图提示词」的全部内容
   - **视频开场提示词**：第九部分「视频开场提示词」的全部内容
3. 同时提取以下信息用于 script 节点的 prompt：
   - **视频方向**（横版/竖版）：从第二部分「项目核心目标」的"横版/竖版"字段获取
   - **视频时长**：从第二部分「项目核心目标」的"视频时长"字段获取
   - **剧名**：从第五部分剧本正文的第一行获取
   - **核心剧情摘要**：将剧本正文的各幕画面描述 + 对白压缩为一段 100-200 字的剧情摘要

**script 节点 prompt 构建规则**：
从剧本文档中提取信息，组装为以下格式的 prompt 传给 script 节点：

```
[核心剧情摘要，100-200字]。[创意调性]风格，[光影色调描述]，[视频时长]秒[横版/竖版]短视频。
```

示例：
```
清晨暖光中，一个27岁中国年轻男子坐在窗边书桌旁，白色T恤，短发，对着手机说'帮我订杯咖啡'，手机屏幕显示AI购物助手推荐咖啡，他笑着点确认。温馨日常风格，晨光暖黄色调，4秒竖版短视频。
```

**完成后**：向用户展示提取结果摘要，确认无误后继续。

### 阶段2：创建 LibTV 项目

> 📋 **第2步：创建 LibTV 项目** — 在 libtv 中新建画布项目

**操作步骤**：
1. 使用剧名作为项目名称
2. 执行命令：
   ```bash
   libtv project create "<剧名>"
   ```
3. 绑定项目到当前目录：
   ```bash
   libtv project use <返回的UUID>
   ```

**完成后**：告知用户项目已创建，显示项目名称和 UUID。

### 阶段3：创建素材区节点

> 📋 **第3步：创建素材区节点** — 在画布左侧创建剧本正文、人物设定、场景设定三个文本节点

按照**横向三列布局**创建素材区节点（左列，x=50）：

| 节点名称 | 类型 | 坐标 | 内容来源 |
|----------|------|------|----------|
| 剧本正文 | text | (50, 100) | 第五部分：完整剧本正文 |
| 人物设定 | text | (50, 450) | 第六部分：人物角色设定 |
| 场景设定 | text | (50, 800) | 第七部分：场景设定 |

**创建命令模板**：
```bash
libtv node --x <x> --y <y> create "<节点名>" -t text -u "content=[\"<提取的内容>\"]"
```

**内容写入注意事项**：
- `-u content=` 的值为 JSON 字符串数组格式 `["内容"]`
- 内容中的换行用 `\n` 表示
- 内容中的双引号用 `\"` 转义
- 内容必须是纯文本，去除 Markdown 的 `#`、`**`、`---` 等标记符号
- 每个节点的内容应包含该部分的完整信息（含"AI生成提示词"字段）

### 阶段4：创建提示词区节点

> 🎨 **第4步：创建提示词区节点** — 在画布中间创建静态封面和视频开场提示词节点

按照**横向三列布局**创建提示词区节点（中列，x=500）：

| 节点名称 | 类型 | 坐标 | 内容来源 |
|----------|------|------|----------|
| 静态封面提示词 | text | (500, 100) | 第八部分：静态封面图提示词 |
| 视频开场提示词 | text | (500, 450) | 第九部分：视频开场提示词 |

**创建命令**：同阶段3的命令模板。

### 阶段5：创建AI生成区节点

> 🤖 **第5步：创建AI生成区节点** — 在画布右侧创建 script 节点

按照**横向三列布局**创建AI生成区节点（右列，x=950）：

| 节点名称 | 类型 | 坐标 | 说明 |
|----------|------|------|------|
| 剧本 | script | (950, 100) | 脚本节点，prompt 来自阶段1的摘要 |

**创建命令**：
```bash
libtv node --x 950 --y 100 create "剧本" -t script --prompt "<阶段1构建的prompt>" --set "model=GVLM 3.1"
```

**注意**：
- `--set model=` 使用模型名字（如 `GVLM 3.1`），不是模型 key（如 `aurora-3-prime`）
- 如果用户有指定其他脚本模型偏好，按用户选择设置
- prompt 中严禁出现 JOY 相关描述

---

### 阶段6：运行脚本生成

> 🤖 **第6步：运行脚本生成** — 触发 script 节点生成分镜行

**操作步骤**：
```bash
libtv node "剧本" --run
```

**等待生成完成**：此操作会调用 GVLM 3.1 模型生成分镜行，每行包含画面描述、角色动作、情绪、光影、音效、对白、imageGenerationPrompt、videoMotionPrompt 等详细字段。等待 CLI 返回结果（通常需要 10-30 秒）。

**完成后**：向用户展示生成结果摘要（生成了几行分镜，每行的大致内容），确认无误后继续。

---

### 阶段7：生成分镜图（触发即止）

> 🤖 **第7步：生成分镜图** — 使用 `libtv script storyboard` 命令触发分镜图生成，触发后立即告知用户正在生成，**不做任何后续检查**

**操作步骤**：

```bash
libtv script storyboard "剧本" -s "model=Lib Navo 2" -s "aspectRatio=16:9" -s "settings={\"quality\":\"1K\",\"ratio\":\"16:9\"}"
```

**命令说明**：
- `libtv script storyboard` 是专用命令，与画布中"生成分镜"功能一致
- 自动在脚本节点**右侧**创建分镜图组（group）和各分镜 image 节点
- 默认按顺序对各分镜图片节点执行生成，无需手动 `--run`
- `-s "model=Lib Navo 2"` 指定图片生成模型
- `-s "aspectRatio=16:9"` 设置画面比例为横版16:9
- `-s "settings={\"quality\":\"1K\",\"ratio\":\"16:9\"}"` 设置画质1K、比例16:9

**aspectRatio 切换**：
- 如果用户选择竖版视频，将 `aspectRatio` 和 `ratio` 改为 `9:16`
- 如果用户选择横版视频，使用 `16:9`（默认）

**⚠️ 触发即止规则**：
- **不设置自定义超时**：使用 Bash 默认超时，不要设置 `timeout` 参数
- **不做结果检查**：命令发送后，只告知用户「分镜图正在 LibTV 后台生成中，请在画布中查看进度」
- **不重试**：即使超时或部分失败，也绝不重新执行 `script storyboard`（该命令是全量重新生成，会覆盖已有图片）
- **不验证**：不检查生成了几张、是否全部成功

## 错误处理

### 登录状态检查

在每个涉及 libtv 操作的阶段开始前，先验证登录状态：
```bash
libtv project list
```
如果返回认证错误，提示用户：
```
⚠️ LibTV 登录已过期，请重新登录：
  方式1：在浏览器中访问 libtv 并授权
  方式2：执行 libtv login web 重新登录
```

### 节点创建失败

如果某个节点创建失败（CLI 返回错误），执行以下操作：
1. 向用户报告具体错误信息
2. 重试一次（可能是临时网络问题）
3. 如果仍然失败，提示用户可在 libtv 画布中手动创建该节点

### 脚本生成超时

如果 `libtv node "剧本" --run` 超过 60 秒未返回：
1. 告知用户脚本生成可能需要较长时间
2. 建议用户在 libtv 画布中查看生成进度
3. 画布搭建的其他步骤可以继续（storyboard 命令依赖 script 节点完成，需等待）

### 分镜图生成（触发即止，不重试）

`libtv script storyboard` 命令触发后即告知用户正在生成，**不做任何等待、检查或重试**：
- 该命令是全量重新生成，重复执行会覆盖已有图片，浪费费用
- 生成进度和结果由用户在 LibTV 画布中自行查看
- 如有分镜图生成失败，用户在画布中单独重新生成即可

## 注意事项

- **禁止自动生成视频**：本 skill 绝不执行任何会触发视频生成的操作
- **禁止修改用户内容**：从剧本文档提取的内容应原样写入 libtv 节点，不做任何修改或增强
- **JOY 隔离**：写入 script 节点的 prompt 中严禁出现 JOY 相关描述
- **坐标一致性**：手动创建的节点坐标严格按照布局方案设置，确保画布整洁
- **CLI 命令准确性**：所有 libtv 命令以 `libtv --help` 和 `libtv <子命令> --help` 的实际输出为准
- **分镜图用 storyboard 命令**：使用 `libtv script storyboard` 命令生成分镜图，不要手动创建 group + image 节点
